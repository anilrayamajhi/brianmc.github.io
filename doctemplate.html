Authentication

All calls to the Authorize.Net API require merchant authentication. If you do not have a sandbox account you can sign up for one really quickly here.

  • Sample

merchantAuthentication

Element Description Format
name Required.
The merchant’s valid API login ID.
Submit the API login ID used to submit transactions.		 Up to 25 characters.
transactionKey Required.
The merchant’s valid transaction key.
Submit the transaction key obtained by the merchant from the Merchant Interface.		 16 characters. 
                <merchantAuthentication>
		           <name>mytestacct</name>
		           <transactionKey>112223344</transactionKey>
	            </merchantAuthentication>
